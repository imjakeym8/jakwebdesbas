About HTML forms

My Emmet Cheat Sheet!

November 21 - November 24, 2022

This will serve as my study diary/blog as I will be learning more about HTML forms by differentiating HTML form attributes, elements, input types, and many more...

Here, the HTML form tells the HTML:

"Get information from this form after a user types his first name, and last name from the text fields."

The HTML form attributes here include:

"action"
- defines a location where the form is specified for,
"target"
- defines in what form displays the response after submitting the form; and,
"method"
- this is the HTTP method the browser uses to submit the form.

for this particular HTML form, it uses:

"#"
- which the form response is specified for the same url where it is submitted;
"_self"
- which loads the response in the same HTML fram as the current one; and,
"GET"
- which data is created to the "action" attribute URI with a '?' as a separator, and URI is sent to the server.

This HTML form is known as a drop-down list. This particular HTML form tells HTML:

"Get the information from the list of coins in a drop-down option a user chooses to swap a coin to."

The HTML elements included in this particular HTML form are:

‹ select › ; and,
‹ option ›

The HTML form attributes here are as listed, respectively:

"id"
- a bookmark for users to jump to or an identifier by which it can be attributed to its corresponding CSS style;
"name"
- tells the name of the HTML element
"value"
- paired with the ‹ option › element; it tells what a choice can be made inside a drop-down list. Multiple options with its own value can be added for more choices to be made.


This is a textbox style HTML form. This HTML form tells HTML:

"Get the information from the answer a user submits on his/her thoughts about cryptocurrency inside a textbox."

The only HTML element included in this particular HTML form is:

‹ textbox ›
- defines a textbox HTML form where characters can be added and submitted.
